# TR TS 032 Email Monitor

Монитор запускается через GitHub Actions раз в день и отправляет письма только по новым документам, которые относятся к `ТР ТС 032/2013`.

## Что уже настроено

- Скрипт мониторинга: `scripts/tr_ts_032_monitor.py`
- Расписание GitHub Actions: `.github/workflows/tr-ts-032-monitor.yml`
- Файл состояния, чтобы не было повторных уведомлений: `.state/tr_ts_032_seen.json`
- Адрес получателя: `igas312@mail.ru`

## Что нужно добавить в GitHub secrets

В GitHub репозитории открыть:
`Settings -> Secrets and variables -> Actions`

Добавить:

- `SMTP_USERNAME`
- `SMTP_PASSWORD`

Сейчас workflow настроен под `smtp.mail.ru` и будет отправлять письма с ящика, указанного в `SMTP_USERNAME`.

## Рекомендация по mail.ru

Лучше использовать пароль приложения, а не обычный пароль от ящика.

## Как проверить вручную

1. Открыть вкладку `Actions` в GitHub.
2. Выбрать workflow `TR TS 032 monitor`.
3. Если хочешь проверить доставку письма сразу, включить `send_test_email`.
4. Нажать `Run workflow`.

## Как работает первый запуск

Первый запуск делает инициализацию:

- забирает текущие документы по `ТР ТС 032/2013`
- записывает их в состояние
- не отправляет старый backlog на email

Начиная со следующего запуска будут приходить только новые документы.
